🔴 PROBLEM REPORTED:
   When registering patients via VAPI agent, all parameters (firstName, lastName,
   email, phone) were showing as "undefined" resulting in 400 Bad Request error

🟢 ROOT CAUSE IDENTIFIED:
   VAPI sends parameters in different nested formats, but endpoint only handled
   1-2 formats. Missing formats → all parameters undefined

🟦 SOLUTION APPLIED:
   ✅ Enhanced register-patient endpoint with 6 payload format support
   ✅ Enhanced find-doctor endpoint with 6 payload format support  
   ✅ Enhanced check-doctor-availability endpoint with 6 payload format support
   ✅ Added support for multiple parameter name variations (camelCase + snake_case)
   ✅ Added comprehensive logging for debugging VAPI payload issues
   ✅ Improved error messages with actual received data

PAYLOAD FORMATS NOW SUPPORTED:
  1. { input: { firstName, ... } }
  2. { parameters: { firstName, ... } }
  3. { toolUse: { input: { firstName, ... } } }
  4. { firstName, ... }
  5. { data: { firstName, ... } }
  6. { messages: [{ content: "..." }] }

PARAMETER NAME VARIATIONS SUPPORTED:
  firstName:   firstName, first_name, FirstName, first, fname
  lastName:    lastName, last_name, LastName, last, lname
  email:       email, userEmail, user_email, patientEmail, patient_email
  phone:       phone, phoneNumber, phone_number, mobile, mobileNumber
  doctorId:    doctorId, doctor_id, doctorID, id
  specialty:   specialization, specialty, speciality, specialist
